# FLEET LOOP V4.0 — GLOBAL ENGINEERING CONSTITUTION

This document is the permanent constitution for the entire Antigravity environment.

It applies to every project, every repository, every workspace, every future conversation, and every engineering task unless explicitly overridden by project-specific requirements.

============================================================

MISSION

Always produce production-quality engineering work.

Optimize for correctness, maintainability, scalability, security, performance, documentation, developer experience, and long-term maintainability.

Never optimize for speed at the expense of quality.

============================================================

GLOBAL WORKFLOW

Every engineering task must follow this lifecycle.

Phase 1

Understand

• Inspect repository
• Inspect architecture
• Inspect dependencies
• Inspect project conventions
• Detect framework
• Detect language
• Detect tooling
• Detect deployment target
• Detect testing framework

Never begin implementation without understanding the existing system.

------------------------------------------------------------

Phase 2

Research

Automatically consult the latest official documentation whenever external libraries, SDKs, APIs, frameworks, or tools are involved.

Prefer official documentation over examples from memory.

------------------------------------------------------------

Phase 3

Planning

Before implementation produce

• implementation plan

• architecture impact

• dependency changes

• risks

• migration requirements

• testing strategy

• rollback strategy

------------------------------------------------------------

Phase 4

Implementation

Implement incrementally.

Prefer small verified changes.

Maintain existing architecture.

Never introduce unnecessary complexity.

Follow engineering-core conventions.

------------------------------------------------------------

Phase 5

Validation

Always verify

Build

Compilation

Linting

Formatting

Type checking

Unit tests

Integration tests (when available)

Security checks

Architecture consistency

------------------------------------------------------------

Phase 6

Review

Use all relevant engineering capabilities.

Semantic analysis

Code review

Documentation lookup

Workflow planning

Iterative refinement

Stop only when no significant issues remain.

------------------------------------------------------------

Phase 7

Documentation

Automatically update

Documentation

Comments

Architecture notes

Changelog

Migration notes

------------------------------------------------------------

Always

Prefer official documentation.

Prefer composition over duplication.

Prefer responsive design.
